2.内部类

内部类：Java中，可以将一个类定义在另一个类里面或者一个方法里面，这样的类叫做内部类。

与之对应，包含内部类的类被称为外部类。

内部类隐藏在外部类之内，更好地实现了信息的隐藏。

内部类的分类：1.成员内部类，2.静态内部类，3.方法内部类，4.匿名内部类

1.成员内部类：

​ （1）内部类相当于外部类的一个成员变量，可以使用任意访问修饰符。

​ （2）内部类中定义的方法可以直接访问外部类中的数据，而不受访问修饰符的影响。

​ （3）外部类不能直接使用内部类的成员和方法，需要借由内部类对象完成。

​ （4）需要通过外部类对象来创建内部类实例。

​ （5）若外部类和内部类具有相同的成员，内部类默认优先访问自己的成员；可以通过”外部类.this.对象成员”以及”外部类.静态成员”的方式访问外部类成员。

​ （6）编译后产生：外部类$内部类.class

2.静态内部类：

​ （1）静态内部类对象可以不依赖于外部类对象，直接创建。

​ （2）静态内部类不能直接访问外部类的非静态成员，但可以通过”外部类对象.成员”的方式访问。

4.接口Interface

Java中只支持单继承，如何解决一个类型中需要兼容多种类型特征的问题？以及多个不同类型具有相同特征的问题呢？

接口：定义了某一批类所需要遵守的规范，接口不关心这些类的内部数据，也不关心这些类里方法的实现细节，它只规定这些类里必须提供某些方法。

修饰接口：public 或者 缺省

接口的继承：可以继承多个父类

若父接口中有同名的方法，子接口不知道用哪个，则需要自己定义一个同名方法自己用

9.多态Polymorphism

面向对象程序设计中，最重要的特征：多态，封装和继承都是为了多态做准备。

多态：意味着允许不同类的对象对统一消息做出不同的响应。

编译时多态：设计时多态，方法重载。

运行时多态：Java程序运行时根据调用该方法的实例的类型来动态决定选择调用哪个方法。（Java中）

必要条件：-满足继承关系。-父类引用指向子类对象。

  • 向上类型转换（Upcast）：将子类型转换为父类型。

格式：父类引用 子类对象

= 隐式/自动类型转换，是小类型到大类型的转换

= 对于向上的类型转换，不需要显式指定，即不需要加上前面的小括号和父类类型名

父类引用指向子类实例，可以调用子类重写父类的方法以及父类派生的方法，但是无法调用子类独有方法。

注意：父类中的静态方法无法被子类重写，所以向上转型之后，只能调用到父类原有的静态方法

  • 向下转型（Downcast）：将父类型转换为子类型

格式：子类引用 父类对象

= 必须强制类型转换，是大类型到小类型的转换

= 必须满足转型条件才能进行强转（相同父类的子类不能强转（相当于老大转成老二，一定有错误））

  • instanceof运算符：返回true/false

格式：对象 instanceof 类名

查看左侧对象是否是右侧类的实例，对于实例类型，自己的引用类型，自己的父类类型均为true

  • 抽象类：不能直接实例化，只能被继承；可以通过向上转型，指向子类实例。

  • 抽象类应用场景：某个父类只是知道其子类应该包含怎样的方法，但是无法准确知道这些子类如何实现这些方法。

  • 抽象方法：不允许包含方法体；子类中需要重写父类的抽象方法，除非将子类也声明为抽象类。

包含抽象方法的类是抽象类，抽象类中可以没有抽象方法。

为什么要定义抽象？ 提醒你这些类必须要有一些方法，要重写这些父类的方法。

14.单例模式

要点：1.某个类只能有1个实例；2.必须自行创建实例；3.必须自行向系统提供这个实例。

实现：1.只提供私有的构造方法；2.含有1个该类的静态static私有对象。（随着类的实例化的产生而产生，无论该类实例了多少个对象，均共享同一个静态成员）3.提供一个静态的公有方法用于创建、获取静态私有对象。

代码实现方案：饿汉式：对象创建过程中实例化；

懒汉式：静态公有方法中实例化。

//饿汉式
//创建对象实例的时候直接初始化  空间换时间  
//加载类时就已经初始化对象 速度快 存在时间较长
//在测试类中调用2次get方法再输出时，两个引用指向同一个实例对象
public class SingletonOne&#123;
    //1.创建类中私有构造
    private SingletonOne()&#123;
        
    &#125;
    //2.创建该类型的私有静态实例
    private static SingletonOne instance = new SingletonOne();
    
    //3.创建公有静态方法返回静态实例对象
    public static SingletonOne getInstance()&#123;
        return instance;
    &#125;
&#125;

//懒汉式
//类内实例对象创建时并不直接初始化，直到第一次调用get方法时，才完成初始化操作
//时间换空间
//在测试类中调用2次get方法再输出时，两个引用指向同一个实例对象
public class SingletonTwo &#123;
    //1、创建私有构造方法
    private SingletonTwo()&#123;
        
    &#125;
    //2、创建静态的该类实例对象  先不进行初始化
    private static SingletonTwo instance=null;
    
    //3、创建开放的静态方法提供实例对象
    public static SingletonTwo getInstance()&#123;
        if(instance==null)
            instance=new SingletonTwo();  //第1次调用时进行实例化，后面的调用都不再实例化
        return instance;
    &#125;
&#125;

饿汉式线程安全，懒汉式存在线程风险：解决：1.同步锁，2.双重校验锁，3.静态内部类，4.枚举。

单例模式优点：1.在内存中只有一个对象，节省内存空间。2.避免频繁的创建销毁对象，提高性能。3.避免对共享资源的多重占用。

缺点：1.扩展比较困难。2.若实例化后的对象长期不利用，系统将默认为垃圾进行回收，造成对象状态丢失。

单例模式使用场景：1.创建对象时占用资源过多，但同时又需要用到该类对象。2.对系统内资源要求统一读写，如读写配置信息。3.当多个实例存在可能引起程序逻辑错误，如号码生成器。

15.异常

异常本质上是程序上的错误。包括编译期间的错误和运行期间的错误。

编译期间的错误编译器会发现并且让你改正，而运行期间的错误编译器就无法发现了，运行期间的错误往往是难以预料的。

17.集合

  • 集合是工具类，可以存储任意数量的具有共同属性的对象。

  • 应用场景：1.无法预测存储数据的数量。2.同时存储具有一对一关系的数据。3.需要进行数据的增删。4.数据重复问题。

    Collection类的对象、Map键值对

  • List（列表）

  • List是元素有序并且可以重复的集合，成为序列。

  • List可以精确地控制每个元素的插入位置，或删除某个位置的元素。

  • List的两个主要实现是ArrayList和LinkedList。

    实现类：

  • ArrayList底层是由数组实现的。

  • 元素有序且可以重复。

  • 动态增长，以满足应用程序的需求。

  • 在列表尾部插入或删除数据非常有效。

  • 更适合查找和更新元素。

  • ArrayList中的元素可以为NULL。

  • Set是元素无序并且不可以重复的集合，被称为集。

    实现类：

  • HashSet是一个重要实现类，称为哈希集。

  • 元素无序并且不可以重复。

  • 只允许一个NULL元素。

  • 具有良好的存取和查找性能。

  • Iterator(迭代器)：Iterator接口可以以统一的方式对各种集合元素进行遍历。

  • hasNext()方法检测集合中是否还有下一个元素。

  • next()方法返回集合中的下一个元素。

    hashCode()，按照一定的规则，将与对象相关的信息映射为一个数值，具体需求具体对待，下图为IDE提供的HashCode()。

    equals()方法判断为true后，hashCode一定为true，但是hashCode()为true时，还需要equals()方法来判断属性是否相等。

  • Map中的数据是以键值对（key-value）的形式存储的。

  • key-value以Entry类型的对象实例存在。

  • 可以通过key值快速地查找value。

  • 一个映射不能包含重复的键。

  • 每个键最多只能映射到一个值。

    实现类：

  • HashMap：基于哈希表的Map接口的实现。

  • 允许使用null值和null键（只能有一个null键）。

  • key键值不允许重复。

  • HashMap中的Entry对象是无序排列的。
