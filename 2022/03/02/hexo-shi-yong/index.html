git与hexo的使用

发表于2022-03-02|更新于2022-03-02
|阅读量:

hexo安装

官网安装node.js

打开命令行(bashshell,gitbash)

注：以下命令均在users目录下使用，均在Windows下使用（其他系统命令或许稍有不同）

clear 清除

npm -v 查看npm包管理器的版本

node -v 查看node的版本

npm install -g cnpm –registry=https://registry.npm.taobao.org 安装镜像源

（-g代表global全局安装）

cnpm 查看cnpm工具

cnpm -v 查看cnpm版本

cnpm install -g hexo-cli 安装hexo博客框架（安装在users的新建文件夹中）（中间若出现难以解决的问题，删除此文件夹重新开始即可）

hexo -v 查看hexo版本

hexo init 生成hexo博客（git bash中使用）（管理员身份使用）

显示Start blogging with Hexo! 说明安装成功

hexo常用指令

hexo s~~启动hexo

hexo n “博客标题”~~new（文章均默认为markdown格式）

（修改可直接用markdown修改后保存）

（也可以用 vim 文件名.md 修改，Esc键+wq+Enter键退出）

hexo clean~~清理缓存

hexo g~~生成文章（generate）

hexo d~~部署至github

hexo部署

为方便打开，部署至github

新建一个仓库，仓库名必须为 你的用户名.github.io

描述可以任意写

npm install hexo-deployer-git –save（Windows下安装git插件）

（若出现错误：ERROR Deployer not found: git使用以上指令修复）

用任意文本编辑器打开blog/_config.yml （例如记事本）

找到# Deployment 下修改为 type: git （注：冒号后的空格不能缺少，若出现错误尝试添加单引号’git’）

添加type同级说明repo

修改为repo: 新仓库的地址 （注）

添加type同级说明branch

修改为branch: master （注）

hexo d~~进行部署（过程需要输入GitHub账号和密码）

hexo d -g~~生成并部署

hexo s~~开始（查看效果，http://localhost:4000查看）

修改config文件里的themes: butterfly

hexo clean -> hexo g -> hexo s -> hexo d

hexo文件说明

source/_posts/~~即为发布的文章

themes~~即为主题

_config.yml~~即为设置文件

git使用

git是版本控制工具

subl 文件名~~用sublime_text打开文件（需要配置sublime_text）

新建本地git仓库Respository：git init（ls -a进行查看是否成功）

git commit~~提交跟踪储存（git commit -a一步到位，第一次需要add）

（注：规范：在sublime中编辑更新说明COMMIT_EDITMSG时，以大写动词开头，会自动弹出编辑窗口）

git add 文件名~~添加想要跟踪的文件，add到暂存区stage之后再git commit到git仓库

git commit -m “更新说明”~~add后直接commit，不打开文本编辑器（-m即massage）

git rm –cached 文件名~~将文件从暂存区删除（不影响本地文件，遇到nothing to commit, working tree clean时，说明已经commit过了，可以尝试使用此命令删除暂存区文件后再次add，commit）

git status~~查看跟踪状态，是否有修改

git log~~查看git仓库工作日志

git diff 版本号1 版本号2~~对比仓库中的不同提交版本（版本号为黄色字体）

git diff~~直接查看工作区和暂存区的区别

git diff –staged~~查看暂存区和仓库的区别

git reset –hard~~将工作区和暂存区的内容修改回仓库中的内容

git branch~~查看分支（master为主干）

git branch 分支名~~创建分支

*~~表示现在处于哪个分支（git status）

git checkout 分支名~~切换到某个分支（注意大小写）

git merge 分支名1 分支名2~~合并两个分支至分支1

（Automatic merge failed; fix conflicts and then commit the result.自动合并可能出现冲突合并失败，要解决冲突）

冲突conflict：两个分支修改了同一个东西为不同版本，例：

<<<<<<< HEAD

version_1.03

=======

version_1.04

<<<<<<< test

手动解决：删去冗余字符，保留想保留的版本，再保存：

version_1.04

git status后提示有未合并的路径

git remote~~查看已经remote的origin

git remote -v~~查看已经remote的origin的详情

git remote add origin_name https://github.com/Ayuhaha/git_intro.git（仓库http地址）~~remote新的origin（git@github.com:Ayuhaha/git_intro.git（仓库ssh地址）也可）

git remote rm origin~~解除原有的连接

git push origin master~~将本地的master推到GitHub仓库的origin（可能会弹出登录github账号密码，注意设置好默认浏览器）

git push

在云端仓库添加新文件：Add file -> create new file -> 编辑标题和内容，Commit new file处为说明

git pull origin masster~~将云端的仓库拉到本地

出现错误：fatal: unable to access ‘https://github.com/Ayuhaha/git_intro.git/': SSL certificate problem: unable to get local issuer certificate时，可尝试：git config –global http.sslVerify false后再次pull

将大神的代码Fork到自己的仓库

rm -rf 仓库名~~删除仓库

可以将仓库导入码云再clone比较快

（git clone 码云的仓库地址）

改为与github关联：进入文件中.git隐藏目录，用文本编辑器打开config，

将远程地址url中的gitee的改为github的，这样就变为了与github关联，偷天换日

git clone 仓库地址 本地目录~~克隆到本地的目录

例：git clone -b master https://github.com/jerryc127/hexo-theme-butterfly.git themes/butterfly

hosts文件位置：C:\Windows\System32\drivers\etc

cmd常用指令(bash shell)DOS命令

pwd~~查看当前路径

ls~~列举当前目录下所有文件及文件夹

dir~~列举当前目录下的文件以及文件夹

ls -l~~查看当前路径下的文件详情

ls -a~~查看文件详情及路径

cd 文件名~~打开文件

cd ..~~返回上一级

cd \ ~~退回根目录

mkdir 文件夹名~~创建新文件夹

md 文件夹名~~创建目录

rd 文件夹名~~删除目录

del 文件名~~删除文件

cat 文件名~~查看文件内容

Tab键~~自动补全文件名

q键~~退出（quit）

nano 文件名~~新建一个文本并编辑或编辑文本（nano为自带的文本编辑器）

ctrl+x~~退出nano

man 方法名~~查看此方法的信息？？？

diff 文件1 文件2~~比较两个文件的不同（原始方式）（注意加文件后缀）

clear~~清空记录

ipconfig/flushdns~~刷新DNS解析缓存

ping 域名~~测试本机至IP主机之间的网络是否通畅

nslookup~~查询域名解析是否正常

ifconfig -a~~查看本机的IP（inet地址）（windows）

ifconfig~~查看本机的IP（ens一栏）（linux）

win键+R键~~打开搜索->cmd

以管理员身份运行cmd命令提示符的方法：1、点击开始菜单，打开所有应用；2、打开windows系统文件夹，右键点击命令提示符；3、在弹出的选项中选择【以管理员身份运行】选项即可。

jar xvf 压缩包.zip~~解压
